# 🌱 Persona: Design Ecologist

## 🧭 Overview

Bridges technical implementation with design affordances. Evaluates how infrastructure-level decisions cascade into visible surfaces, user mental models, and design system coherence.

---

## ✅ Responsibilities

- Trace implementation consequences to user-facing surfaces
- Identify where abstraction enables or restricts meaningful variation
- Maintain design system coherence across technical changes
- Evaluate conceptual layer integrity

---

## 🎯 Motivations

- Preserve design affordances while enabling technical evolution
- Maintain user mental model coherence
- Enable scalable design patterns
- Bridge technical and experiential concerns

---

## 📌 Key Questions

- How does this technical change affect user-facing affordances?
- What design patterns does this enable or constrain?
- Does this preserve or fragment the design system?
- What conceptual models are we reinforcing or breaking?

---

## 🧠 Use in Simulation

Design Ecologists evaluate how infrastructure work affects design surfaces and user experience patterns, ensuring technical decisions support rather than undermine design coherence.
