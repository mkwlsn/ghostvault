# 🪄 ghostOS DocOps Rituals

this folder contains **rituals** used for persona simulation, planning, and feature evaluation in the ghostOS project.  
rituals are markdown-based roleplay templates designed for AI agents and human collaborators alike.

---

## 📁 included rituals

each `.md` file in this directory corresponds to a specific **persona**:

| Persona           | Purpose                                                                 |
|------------------|-------------------------------------------------------------------------|
| product_owner     | prioritization, scope management, milestone definition                 |
| systems_engineer  | architecture planning, dependency resolution, risk analysis            |
| communications_architect | language clarity, audience tuning, messaging resonance       |
| systems_philosopher | foundational alignment, semiotic integrity, worldview consistency   |
| contributor       | ease-of-use checks, modularity, learning curve, git ergonomics         |

---

## 🛠 how to use

1. copy/paste the contents of any ritual into your LLM interface (e.g. ChatGPT, Claude, OpenWebUI).
2. optionally inject a PRD or feature doc as a system message or prefix.
3. let the model simulate the persona and evaluate the feature or decision space.

---

## 🤝 how to contribute

- create a new `.md` file named after the persona or ritual (`legal_counsel.md`, `ux_researcher.md`, etc).
- follow the structure used in existing files: intro, context, behavioral guidance.
- submit a pull request with your addition.

---

## 🧠 why this exists

ghostOS is being built as both a product and a system.  
to simulate real-world complexity and scale, we use **rituals** to bring in structured perspectives.  
this makes roadmap planning, feature validation, and design philosophy more robust.
